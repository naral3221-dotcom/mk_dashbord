# Prompt Guide - 세션 시작 방법

## 🚀 세션 시작 시 (필수)

### 방법 1: 커맨드 사용
```
/start
```

### 방법 2: 자연어
```
세션 시작
```
```
컨텍스트 복원해줘
```
```
이어서 작업하자
```

### 결과
오케스트레이터가 자동으로:
1. CONTEXT.md 읽고 상태 복원
2. ROADMAP.md 읽고 현재 스프린트 확인
3. 오늘 로그 파일 확인/생성
4. 현재 상태 보고

---

## 📋 일반 작업 요청 (세션 중)

세션이 이미 시작된 상태에서는 그냥 요청:

```
다음 태스크 진행해
```
```
Campaign 엔티티 구현해줘
```
```
Sprint 0 계속 진행
```

---

## 🆚 언제 /start를 써야 하나?

| 상황 | /start 필요? |
|------|-------------|
| 새 대화 시작 | ✅ 필수 |
| 브라우저 새로고침 후 | ✅ 필수 |
| 오랜 시간 후 재개 | ✅ 필수 |
| 같은 세션 내 연속 작업 | ❌ 불필요 |
| 방금 작업 완료 후 다음 작업 | ❌ 불필요 |

---

## 📝 프롬프트 예시

### 세션 시작 + 작업 지시
```
/start 하고 Sprint 0 계속 진행해줘
```

### 특정 태스크 지정
```
/start

0.3 Clean Architecture 폴더 구조 만들어줘
```

### 상태만 확인
```
/start

현재 상태만 알려줘
```
